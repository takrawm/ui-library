# ドキュメント同期フロー

## 概要

プロジェクトの標準ルールは、以下の2つのドキュメントで管理されています。
内容を常に一致させるため、どちらかを更新した場合はもう一方も必ず更新してください。

| ドキュメント | 用途 |
|-------------|------|
| `CLAUDE.md` | Claude（Anthropic）での開発時に参照されるプロジェクトメモリ |
| `.cursor/rules/project-memory.mdc` | Cursorでの開発時に適用されるルール（alwaysApply: true） |

## 同期対象の関係

```
CLAUDE.md  ←── 双方向同期 ──→  .cursor/rules/project-memory.mdc
```

## 更新フロー

### パターンA：CLAUDE.md を更新した場合

1. `CLAUDE.md` を編集・保存する
2. **直ちに** `.cursor/rules/project-memory.mdc` を更新する
3. 更新内容：
   - フロントマター（`---` で囲まれたYAML）はそのまま維持
   - 先頭の「ドキュメント同期」の注意書きはそのまま維持
   - 本文（`## 概要` 以降）を `CLAUDE.md` の内容と一致させる

### パターンB：.cursor/rules/project-memory.mdc を更新した場合

1. `.cursor/rules/project-memory.mdc` を編集・保存する
2. **直ちに** `CLAUDE.md` を更新する
3. 更新内容：
   - `project-memory.mdc` の本文（`## 概要` 以降、フロントマターと同期注意書きを除く）を `CLAUDE.md` に反映する

## 更新時の注意点

### project-memory.mdc の構造

`.cursor/rules/project-memory.mdc` は以下の構成です：

```
---
description: ...
alwaysApply: true
---

# プロジェクトメモリ（Cursor用）

> **ドキュメント同期：** ...（このブロックは維持）

## 概要
（ここ以降が CLAUDE.md と同期する本文）
```

- **維持する部分**：フロントマター、タイトル、同期注意書き
- **同期する部分**：`## 概要` 以降の本文

### CLAUDE.md の構造

`CLAUDE.md` は以下の構成です：

```
# CLAUDE.md (プロジェクトメモリ)

## 概要
（本文）
```

- 先頭の `# CLAUDE.md (プロジェクトメモリ)` は維持
- `## 概要` 以降が `project-memory.mdc` の本文と一致する

## チェックリスト

更新後、以下を確認してください：

- [ ] 両方のドキュメントで「概要」「プロジェクト構造」「開発プロセス」「ドキュメント管理の原則」「図表・ダイアグラム」「注意事項」の内容が一致している
- [ ] `project-memory.mdc` のフロントマターが正しく設定されている（`alwaysApply: true`）
- [ ] 両方のドキュメントに「ドキュメント同期」への言及がある（`CLAUDE.md` は本ドキュメントへの参照を追加すること）

## AIエージェントへの指示

AIエージェント（Claude、Cursor）がどちらかのドキュメントを更新する場合：

1. 更新対象のドキュメントを編集する
2. **同じ編集セッション内で**もう一方のドキュメントも更新する
3. 変更内容が両方に反映されていることを確認する
