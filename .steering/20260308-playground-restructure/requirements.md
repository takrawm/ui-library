# 要求内容: プレイグラウンド画面の再構成

## 背景

プレイグラウンドの目的は、shadcn/ui のコンポーネントを組み合わせて実用的な UI パターンを構築・蓄積することにある。個別コンポーネントのバリエーション表示（ショーケース）は shadcn/ui 公式サイトで代替可能であり、このアプリケーションで保持する必要がない。

## 変更内容

### 1. コンポーネントショーケースの削除

- `src/app/playground/components/` ディレクトリ配下の全ページを削除
  - `button/page.tsx`
  - `card/page.tsx`
  - `badge/page.tsx`
  - `input/page.tsx`
  - `table/page.tsx`
  - `accordion/page.tsx`
- プレイグラウンドトップページ（`page.tsx`）から「コンポーネントショーケース」セクションを削除
- ファイルシステム走査の `components` 関連ロジックを削除

### 2. プレイグラウンドトップページのレイアウト変更

現在のレイアウト:
```
[コンポーネントショーケース（6件カード）]  ← 削除
[組み合わせデモ（3件カード + 新規作成）]
[ComponentBrowser（一体型：コンポーネント一覧 + プロンプトテンプレート）]
```

新しいレイアウト:
```
[ComponentBrowser（コンポーネント一覧のみ）]
[プロンプトテンプレート（独立表示）]
[組み合わせデモ（3件カード + 新規作成）]
```

### 3. ComponentBrowser からプロンプトテンプレートを分離

- ComponentBrowser はコンポーネント一覧（検索・フィルタ・インストール状態表示）のみを担当
- プロンプトテンプレートは独立したセクションとしてページに配置
- 両者はページ上部に配置し、組み合わせデモはその下に表示

## 残すもの

- **ComponentBrowser**: shadcn/ui 全コンポーネント一覧（検索・カテゴリフィルタ・インストール状態）
- **プロンプトテンプレート**: デモ生成用テンプレート（独立表示）
- **組み合わせデモ**: `playground/demos/` 配下のデモページ一覧
- **デモページ本体**: `stat-cards`, `login-form`, `three-column-layout`

## 受け入れ条件

- [ ] `playground/components/` ディレクトリが削除されている
- [ ] プレイグラウンドトップページに「コンポーネントショーケース」セクションが存在しない
- [ ] ComponentBrowser がページ上部に表示される（コンポーネント一覧のみ）
- [ ] プロンプトテンプレートが ComponentBrowser の下に独立セクションとして表示される
- [ ] 組み合わせデモがプロンプトテンプレートの下に表示される
- [ ] `pnpm build` がエラーなく通る
- [ ] `pnpm lint` がエラーなく通る
