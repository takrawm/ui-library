# 要求内容：インタラクティブデモプレイグラウンド

## 概要

UI コンポーネントライブラリに、Claude Code / Cursor でコンポーネントデモを生成し、Next.js dev server のホットリロードで即座にプレビュー・操作できるインタラクティブなプレイグラウンド機能を追加する。

## 背景と目的

現在のコンポーネントカタログ（`src/app/page.tsx`）は静的な一覧表示に留まっており、コンポーネントの組み合わせや動的な振る舞い（クリック、ドラッグ&ドロップ、フォーム入力など）を試すには、コードを書いて `pnpm dev` で確認する必要がある。

本機能により、以下を実現する：

1. **自然言語でデモを生成**: Claude Code / Cursor に自然言語で指示し、ライブラリ内のコンポーネントを組み合わせたデモ UI を自動生成する
2. **会話的にデモを改善**: 生成されたデモに対して、Claude Code / Cursor 上で追加の自然言語プロンプトで修正・改善を指示できる
3. **インタラクティブな動作確認**: Next.js dev server のホットリロードにより、ブラウザ上でクリック、ドラッグ&ドロップ、キーボード操作など、実際の動作を即座に確認できる
4. **デモの永続化**: デモファイルがファイルシステムに保存され、Git でバージョン管理可能

## 機能要件

### FR-P1: プレイグラウンド UI

Web アプリケーション上にプレイグラウンド画面を提供する。

- **デモ一覧ページ（`/playground`）**: 作成済みデモの一覧をカード形式で表示
  - デモ名、説明、作成日時を表示
  - カードクリックで個別デモページに遷移
  - 「新しいデモ」カードで Claude Code / Cursor での生成手順を案内
- **個別デモページ（`/playground/demos/[slug]`）**: デモコンポーネントがフルサイズでレンダリングされ、実際に操作可能なページ
  - クリック、ホバー、フォーカス、ドラッグ&ドロップなどのインタラクションが動作する
  - ブラウザのリサイズでレスポンシブ表示を確認可能
  - 「一覧に戻る」リンク
- **コンポーネント参照パネル**: 利用可能なコンポーネントの一覧を折りたたみ可能なパネルで表示
  - Claude Code / Cursor でプロンプトを書く際の参照用

### FR-P2: デモ生成ワークフロー

Claude Code / Cursor を使ってコンポーネントデモを生成する。

- ユーザーが Claude Code / Cursor 上で自然言語プロンプトを入力してデモコードを生成
  - 例: 「3つのカードをドラッグ&ドロップで並べ替えられるリストを `src/app/playground/demos/sortable-list/page.tsx` に作成して」
  - 例: 「ログインフォームをバリデーション付きで作って」
- ライブラリ内の既存コンポーネント（`src/components/` 配下）をベースに組み合わせる
- shadcn/ui のベースコンポーネント（`src/components/ui/`）も利用可能
- 生成結果は Next.js ページコンポーネント（TSX）として出力される
- `pnpm dev` のホットリロードで即座にブラウザ上にプレビュー表示される

### FR-P3: 会話的なデモ改善

生成済みのデモに対して、Claude Code / Cursor 上で追加プロンプトにより修正・改善を指示できる。

- 現在のデモファイルに対して、Claude Code / Cursor で追加の自然言語プロンプトを送信できる
  - 例: 「カードの色を青系に変更して」
  - 例: 「テーブルにソート機能を追加して」
  - 例: 「レイアウトを2カラムから3カラムに変えて」
- Claude Code / Cursor の会話機能により、文脈を保持した差分的な修正が可能
- ファイル保存時にホットリロードでプレビューが即座に更新される

### FR-P4: デモの永続化と管理

作成したデモをファイルシステムで管理する。

- デモファイルは `src/app/playground/demos/[slug]/page.tsx` に保存
- ファイルシステムの走査でデモ一覧を動的生成（DB 不要）
- Git でバージョン管理可能（変更履歴の追跡）
- 不要なデモはディレクトリごと削除

### FR-P5: コンポーネント参照

プレイグラウンドからライブラリ内のコンポーネントを参照できる。

- 利用可能なコンポーネントの一覧表示
- 各コンポーネントの説明と import パスの表示
- 折りたたみ可能なパネル

## 非機能要件

### NFR-P1: 技術スタック

既存の技術スタックのみを使用する（追加パッケージなし）。

- **フレームワーク**: Next.js (App Router) — 既存プロジェクトの一部として構築
- **UI**: shadcn/ui ベースコンポーネント + ライブラリ内カスタムコンポーネント
- **言語**: TypeScript + React（TSX）
- **プレビュー**: Next.js Dev Server（ホットリロード）
- **コード生成**: Claude Code / Cursor（Claude.ai Max プラン枠内で利用）
- **データ永続化**: ファイルシステム（DB 不要）

### NFR-P2: パフォーマンス

- Next.js dev server のホットリロードにより、ファイル変更が即座にブラウザに反映される
- デモ一覧ページの読み込みは実用的な速度であること
- デモ上でのインタラクションはスムーズであること（フレームレート低下なし）

### NFR-P3: 開発体験

- Tailwind CSS v4 がそのまま動作する（CDN 版 v3 の制約なし）
- ライブラリコンポーネントを `@/components/` から直接 import できる
- 追加の環境変数や設定ファイルは不要

## ユーザーストーリー

### US-P1: デモの自然言語生成

開発者として、Claude Code / Cursor に「統計カードを4枚並べたダッシュボードのヘッダー部分を作って」と指示すると、ライブラリ内のコンポーネントを使ったデモが `src/app/playground/demos/` に自動生成される。`pnpm dev` で起動したローカルサーバーでブラウザから即座に確認できる。

### US-P2: 会話的なデモ改善

開発者として、ブラウザでデモを確認しながら、Claude Code / Cursor 上で「カードの間隔をもう少し広げて」「ヘッダーにアイコンを追加して」と追加指示を出すと、ファイルが修正されてホットリロードでプレビューが更新される。

### US-P3: インタラクティブな動作確認

開発者として、ブラウザ上で生成されたデモのドラッグ&ドロップやボタンクリックなどの操作を実際に試すことができる。Next.js ページとして直接レンダリングされるため、すべてのインタラクションがフル機能で動作する。

### US-P4: デモの一覧確認

開発者として、`/playground` にアクセスすると、これまで作成したデモの一覧が表示される。カードをクリックすると個別のデモページに遷移し、デモを操作できる。

## 受け入れ条件

### AC-P1: プレイグラウンド UI

- [ ] プレイグラウンドページ（`/playground`）がアクセス可能
- [ ] デモ一覧がカード形式で表示される
- [ ] カードクリックで個別デモページに遷移できる
- [ ] 個別デモページでデモがフルサイズ表示される
- [ ] コンポーネント参照パネルが表示される

### AC-P2: デモ生成と改善

- [ ] Claude Code / Cursor でデモファイルを生成できる
- [ ] 生成されたデモがデモ一覧に表示される
- [ ] ホットリロードでプレビューが即座に更新される
- [ ] デモ上でインタラクション（クリック、D&D 等）が動作する

### AC-P3: 品質基準

- [ ] `pnpm build` がエラーなく通る
- [ ] `pnpm lint` がエラーなく通る
- [ ] TypeScript のエラーが 0 件

## 決定済み事項

| 項目 | 決定内容 | 理由 |
|---|---|---|
| コード生成 | Claude Code / Cursor | Claude.ai Max プランの枠内で利用可能。追加 API コストなし |
| プレビュー | Next.js Dev Server（ホットリロード） | ライブラリコンポーネントの直接 import、Tailwind v4 完全対応 |
| データ永続化 | ファイルシステム | DB 不要でシンプル。Git でバージョン管理可能 |

## 設計変更の経緯

当初は Claude API（`@anthropic-ai/sdk`）+ Sandpack + SQLite/Prisma の構成を検討していたが、以下の理由でアプローチを変更：

1. **コスト**: Claude API は従量課金であり、Claude.ai Max プランの枠は使えない
2. **Tailwind 互換性**: Sandpack 内では CDN 版 Tailwind v3 しか使えず、v4 固有機能が利用不可
3. **シンプルさ**: 追加パッケージ・DB・API Route が不要になり、設計が大幅に簡素化

## 制約事項

- 既存のコンポーネントカタログ機能（`src/app/page.tsx`）との共存を維持する
- 既存のコンポーネント設計原則（合成パターン、アクセシビリティ等）を崩さない
- プレイグラウンド機能の追加により、既存の `pnpm build` / `pnpm lint` が壊れないこと
- デモ生成には Claude Code または Cursor が必要（Web UI 単体では生成不可）
- デモのプレビューには `pnpm dev` の起動が必要
